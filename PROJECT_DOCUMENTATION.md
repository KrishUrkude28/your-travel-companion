# TravelSathi AI - Comprehensive Project Documentation

Welcome to the central documentation file for **TravelSathi AI**. This document contains all the details, architecture, and features of your project so you can easily understand and manage it.

---

## 🚀 Project Overview
**TravelSathi AI** is a modern, intelligent travel companion platform. It allows users to browse curated travel packages, generate highly customized day-by-day itineraries using AI, track travel budgets, and seamlessly book their trips.

### 🛠️ Tech Stack
- **Frontend Framework**: React (with Vite for fast bundling)
- **Styling**: Tailwind CSS & generic CSS variables
- **Animations**: Framer Motion
- **Icons**: Lucide React
- **Backend & Database**: Supabase (PostgreSQL, Auth, Storage)
- **AI Integration**: Groq API (running Llama-3 70B for fast generation)
- **Maps**: Keyless Google Maps Embeds
- **PDF Generation**: `html2canvas` and `jsPDF`

## 🗄️ Database Schema (Supabase)

All application data is securely stored in Supabase. The database uses **Row-Level Security (RLS)**, ensuring users can only access their own private data.

### 1. Tables
- **`profiles`**: Created automatically when a user signs up. Stores their full name, avatar, and saved travel preferences (e.g., Adventure, Culture).
- **`wishlists`**: Stores packages that the user has "liked" or saved for later.
- **`trip_plans`**: Stores the custom AI itineraries generated by the user in the Trip Planner.
- **`bookings`**: Stores actual booking requests. Handles both predefined packages and custom AI itineraries.
- **`payments`**: Stores records of simulated payment transactions linked to a booking.
- **`reviews`**: Stores user testimonials displayed on the landing page.

### 2. Storage Buckets
- **`itineraries`**: A public storage bucket. When a user clicks "Share PDF Link", the generated PDF is uploaded here so it can be shared via WhatsApp.

## ✨ Core Features Explained

### 1. AI Trip Planner (`TripPlanner.tsx`)
Users input their destination, duration, budget, and interests. The app contacts the Groq API to generate a structured JSON itinerary.
- **Google Maps**: Automatically embeds a map of the destination.
- **Budget Tracker**: Calculates if the AI plan fits the user's budget.
- **Book this Itinerary**: Users can instantly book the AI's suggestion using the simulated payment gateway.

### 2. PDF Export & Sharing (`itineraryExport.ts`)
Instead of a simple screenshot, the app dynamically creates a hidden, professional document template equipped with the official TravelSathi logo and a dotted pattern background.
- **Download PDF**: Saves the file locally to the user's device.
- **Share on WhatsApp**: Uploads the PDF to the Supabase `itineraries` bucket and generates a WhatsApp message with a direct link.

### 3. Payment Gateway (`PaymentModal.tsx`)
A simulated, highly polished payment modal. It mimics a real credit card checkout (like RazorPay/Stripe).
- Once the user "pays", the app inserts the record into both the `bookings` and `payments` database tables, and sets the booking status to `confirmed`.

### 4. Smart Destination Search (`DestinationAutocomplete.tsx`)
Uses a rapid geocoding API to suggest real cities and countries as the user types, prioritizing tourist hubs and major locations over minor streets.

## 🔑 Environment Variables
To run this project, your `.env` file must contain:

```env
# Supabase Configuration
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key

# Groq API (For AI Itinerary Generation)
VITE_GROQ_API_KEY=your_groq_key

# RapidAPI (For Destination Autocomplete & Weather)
VITE_RAPID_API_KEY=your_rapidapi_key
```

## 🛠️ How to Manage the Backend
As the Admin/Owner, you do not need a custom admin panel to manage the app. 
1. Log in to [Supabase](https://supabase.com/dashboard).
2. Go to the **Table Editor**.
3. Here, you have "Superuser" access. You can view every user's bookings, payments, and generated itineraries. You can manually edit or delete any row.
4. If you ever need to reset the database, simply run the code inside `supabase_migration.sql` in the Supabase SQL Editor.
